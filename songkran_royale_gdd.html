Songkran Royale

Real-time PvP water gun battle — browser game concept

Core loop

ผู้เล่น เลือก avatar (หญิง / ชาย / LGBTQ) รับบท "นักท่องเที่ยวต่างชาติ" ที่มาเยือนไทย → เข้า lobby → จับคู่แบบ real-time → ฉีดน้ำสู้กัน → ใครถังน้ำหมดก่อนแพ้ → คนสุดท้ายที่ยืนอยู่ได้ชื่อว่า จ้าวแห่งสงกรานต์

Gameplay pillars

Move + Aim + Shoot

WASD เดิน / เมาส์เล็ง / คลิกยิงน้ำ (top-down view)

Water tank system

น้ำมีจำกัด ต้องวิ่งหาถังเติม ทำให้เกิด zone control

Wet meter

โดนน้ำสะสม → เปียก 100% = หมดสิทธิ์ round

Map hazard

สระน้ำ / ถนน / ขบวนรถที่สาดน้ำ — ธีมถนนเชียงใหม่

Character system

3 identity preset

หญิง / ชาย / LGBTQ — เลือกก่อนเข้าเกม

Nationality skin

เลือกธง → เสื้อเปลี่ยนสีธงชาติ (viral hook)

Weapon skin

ปืนฉีดน้ำ / ขันน้ำ / สายยาง — unlock ด้วย win streak

Victory pose

animation พิเศษเมื่อเป็นจ้าวแห่งสงกรานต์

Viral hooks

Share link

ชวนเพื่อนด้วย URL เดียว เปิด browser เล่นได้เลย ไม่ต้อง install

Result card

จบเกมได้ภาพ "จ้าวแห่งสงกรานต์" พร้อมสถิติ — แชร์ได้ทันที

Room code

สร้าง room ส่วนตัว 6 ตัวอักษร เล่นกับเพื่อนได้เลย

Stack แนะนำ สำหรับ real-time PvP บน browser

Renderer

Phaser 3 แนะนำ

WebGL + Canvas fallback, physics ในตัว, particle system, tilemap — community ใหญ่ที่สุด เหมาะ top-down 2D สุดๆ

Multiplayer

Colyseus แนะนำ

Game server framework สำหรับ Node.js โดยเฉพาะ — จัดการ room, state sync, matchmaking ได้ครบ ไม่ต้องเขียน socket logic เอง

Transport

WebSocket (built-in Colyseus)

Latency ต่ำ, browser รองรับ 100% — เหมาะกว่า HTTP polling สำหรับ real-time PvP

Backend

Node.js + TypeScript

Share type definition ระหว่าง client กับ server ได้เลย — ลด bug ระหว่าง sync state

Deploy

Vercel (client) + Railway / Fly.io (server)

Client เป็น static HTML/JS deploy Vercel ฟรี, Server ต้องการ persistent WebSocket — Railway หรือ Fly.io รองรับได้ดี

Assets

Aseprite (pixel art) + Tiled (map)

Tiled export เป็น JSON Phaser อ่านได้โดยตรง — workflow เร็วมาก

ทำไม Colyseus ไม่ใช่ Socket.io

Socket.io เป็น general purpose — ต้องเขียน room management, state delta, reconnection logic เอง. Colyseus สร้างมาเพื่อเกมโดยเฉพาะ มี schema-based state sync ที่ส่งแค่ delta ทำให้ bandwidth ต่ำ

ข้อควรระวัง latency

Real-time PvP ต้องการ authoritative server — ทุก action validate บน server ก่อน. ถ้า server อยู่ต่างประเทศ latency สูง ควรเลือก region ที่ใกล้กับ user หลัก (SEA)

System architecture — real-time PvP

Client (Phaser 3)

Input Manager

WASD + mouse → แปลงเป็น intent

↓

Colyseus Client SDK

ส่ง intent → รับ state delta

↓

Reconciliation Layer

interpolate positions + predict

↓

Phaser Scene

render sprites, particles, UI

WebSocket
⟷

Server (Colyseus)

Matchmaking Room

รอผู้เล่น / สร้าง game room

↓

Game Room (Schema)

จัดการ state ของห้องเกม

↓

Physics / Collision

server-side validation

↓

State Broadcast

ส่ง delta ทุก tick (20Hz)

State schema (Colyseus)

GameState {
  players: MapSchema<PlayerState>
  waterTanks: ArraySchema<TankState>
  phase: "lobby" | "playing" | "ended"
  timeLeft: number
}

PlayerState {
  x, y, angle, waterLevel, wetMeter
  character: "female" | "male" | "lgbtq"
  nationality, isAlive, score
}

Development phases

1

Prototype — single player ~1–2 wks

Phaser 3 setup → top-down character เดินได้ → ยิงน้ำ → wet meter → map เบื้องต้น — ทดสอบ feel ของเกมก่อน

2

Local multiplayer — 2P same machine ~1 wk

WASD + Arrow keys → 2 ผู้เล่นบนหน้าจอเดียว → ทดสอบ game balance และ collision

3

Online multiplayer — Colyseus ~2–3 wks

ติดตั้ง Colyseus server → ย้าย game logic ขึ้น server → WebSocket sync → room + matchmaking → deploy Railway

4

Polish + viral features ~1–2 wks

Character selection UI → share link → result card รูปภาพ → sound effect + BGM สงกรานต์ → mobile touch controls

ยาก / เสี่ยงที่สุด

Phase 3 คือจุดที่ยากที่สุด โดยเฉพาะ lag compensation และ cheat prevention — แนะนำให้ทำ authoritative server ตั้งแต่ต้น อย่า client-side physics ก่อนแล้วค่อยย้าย จะงานเพิ่มมาก
